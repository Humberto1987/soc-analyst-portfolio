# Virtualisation Manager Uptime Review

## Objective
Identify the virtual machine running for the longest time and assess system health.

## Steps Taken
- Accessed the Virtualisation Manager dashboard  
- Reviewed VM list and uptime values  
- Identified system errors (Mail-SERVER failure)  
- Verified resource usage and machine states  

## Findings
- Longest-running VM: `monitoring-sys`
- One machine was in a failed state (Mail-SERVER)
- No critical resource exhaustion detected

## Conclusion
This exercise helped me understand how SOC analysts monitor infrastructure health, detect anomalies, and identify systems requiring attention.
